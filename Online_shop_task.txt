Интернет-магазин

Общее:

- Spring MVC приложение по покупке любых товаров;
- Описание Spring-сущностей: Annotation- или Java-подход;
- Java 8;
- Циклические структуры заменять возможностями Stream API;
- Работа приложения на Apache Tomcat 9;
- Темплейт – Thymeleaf;
- Должны быть: контроллеры, слой сервисов, слой ДАО;
- База данных, например, Н2;
- Юнит и интегрейшн тесты обязательны;
- Предусмотреть собственную иерархию исключений;
- Предусмотреть логгирование событий в файл: лог-ин / лог-аут / регистрация пользователей, добавление в корзину и оплата товара, добавление / удаление / обновление товара в магазин;
- Все литералы, используемые в проложении, на английском языке.

Функционал:

- Возможность log in, log out (Spring security на выбор, можно через сессии);
- Только залогиненый пользователь имеет доступ к магазину;
- Время жизни сессии – 3 минуты;
- Если пользователь не залогинен отображать главную страницу (с формой для логина), если залогинен то открывать главную страницу с товарами;
- В случае любой ошибки открывать отдельную страницу с ошибкой в которой есть ссылка на главную страницу;
- Добавить функционал корзины (товары можно удалять и добавлять), добавление товара в корзину не уменьшает количество товаров на складе;
- Систему оплаты сделать любой. Например, просто кнопка оплатить и уменьшить количество товара на складе;
- Каждый товар имеет название, картинку, цену и количество на складе;
- При выборе товара добавить возможность просмотра его описания;
- Админ может добавлять/удалять/обновлять товары;
- Пользовательское меню по категориям товаров.
П.С. Упор делать не на спецэффектах UI, а на качестве Java-кода.
